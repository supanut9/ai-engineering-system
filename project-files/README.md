# README.md

## Purpose

The main entrypoint document for a repository.

## Should Exist

Yes, for every real project.

## Typical Contents

Use a GitHub-friendly structure with short, scan-friendly sections.

Recommended sections:

1. Project name
2. Short description
3. Status
4. Tech stack
5. Prerequisites
6. Local setup
7. Run commands
8. Test commands
9. Environment variables
10. Important docs
11. Deployment notes if relevant

## GitHub Notes

- keep headings clear because GitHub generates a table of contents from section
  headings
- prefer repository-root placement
- use relative links to `docs/` and related files

## Naming

Use `README.md` in uppercase.
