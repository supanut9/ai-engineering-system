# CONTRIBUTING.md

## Purpose

Explains how contributors should work in the repository.

## Should Exist

Yes, for team or multi-agent projects.

## Typical Contents

Use a GitHub-friendly contribution guide with practical contributor steps.

Recommended sections:

1. Scope
2. How to get started
3. Branch workflow
4. Commit conventions
5. Testing expectations
6. Documentation expectations
7. Pull request expectations
8. Review and merge expectations

## Suggested Outline

```md
# Contributing

## Scope

Who should use this guide and what kinds of contributions are welcome.

## Getting Started

- clone the repo
- install dependencies
- configure environment variables

## Branch Workflow

- create a feature branch
- keep changes scoped

## Commit Conventions

- use the project's commit format

## Testing

- run required checks before opening a PR

## Documentation

- update docs when behavior changes

## Pull Requests

- describe the change clearly
- include screenshots or examples when useful
```

## GitHub Notes

- GitHub surfaces `CONTRIBUTING.md` in the repository `Contribute` flow
- place it in the repository root unless there is a specific reason to keep it
  under `.github/` or `docs/`

## Naming

Use `CONTRIBUTING.md` in uppercase.
