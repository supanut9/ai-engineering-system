# functional specification — hello-todo-react-native-expo

Behavioral contract for each screen and interaction. Supplements the acceptance criteria
in `../requirements/acceptance-criteria.md`. Implementation must match this exactly.

References:
- data model and AsyncStorage schema: `data-contract.md`
- screen layouts and navigation: `screens.md`

---

## common behavior

### title validation

Applied on the add screen before calling the service:

- title must be present and non-empty after trimming leading/trailing whitespace.
  Whitespace-only strings (e.g., `"   "`) are rejected.
- trimmed title length must be ≤ 200 characters.
- the trimmed value is what is stored, not the raw input.
- invalid titles show an inline error message and do not call the service or dismiss the
  modal.

### id format

IDs are generated by the service layer using a simple timestamp-plus-random approach:
`Date.now().toString(36) + Math.random().toString(36).slice(2)`. Callers must treat IDs
as opaque strings.

### async storage contract

Every mutation (create, update, delete) calls `lib/storage.saveTodos(todos)` before
returning. `lib/storage.loadTodos()` is called once on mount by the `useTodos` hook.
See `data-contract.md` for the AsyncStorage key and serialization format.

## list screen (`app/index.tsx`)

### mount behavior

- call `useTodos()`, which on first mount loads todos from AsyncStorage via
  `lib/storage.loadTodos()` and populates the in-memory repo.
- while loading, display a loading indicator (ActivityIndicator or equivalent).
- once loaded, render the todo list or the empty state.

### empty state

- when `todos.length === 0`, render the empty-state message "No todos yet".
- do not render the todo list component when the list is empty.

### todo item rendering

Each item in the list renders:
- a completion toggle (e.g., `Pressable` checkbox)
- the todo title
- a delete control (e.g., `Pressable` with a delete label or icon)

Completed items have a visual differentiation (e.g., `textDecorationLine: 'line-through'`
on the title).

### navigation to add screen

A button (e.g., "+ Add" or a floating action button) navigates to `app/add.tsx` as a
modal route using `router.push('/add')` from Expo Router.

### toggle completion

Tapping the completion toggle calls `useTodos().toggleTodo(id)`. The hook calls
`services/todos.toggleTodo(id)`, which flips `completed`, updates `updated_at`, persists
via `lib/storage.saveTodos`, and returns the updated list. The list re-renders
immediately; no loading state is required for this synchronous-ish mutation.

### delete a todo

Tapping the delete control calls `useTodos().deleteTodo(id)`. The hook calls
`services/todos.deleteTodo(id)`, removes the item from the repo, persists via
`lib/storage.saveTodos`, and returns the updated list. The list re-renders immediately.

## add screen (`app/add.tsx`)

### input

A single text input for the todo title. Initial value is empty.

### validation (client-side, before service call)

- title is trimmed before validation and before passing to the service.
- empty or whitespace-only title: display "Title is required" inline; do not submit.
- title exceeds 200 characters after trim: display "Title must be 200 characters or fewer"
  inline; do not submit.

### submission

On valid title:
1. Call `services/todos.createTodo({ title: trimmedTitle })`.
2. The service generates an id, sets `completed: false`, sets `created_at` and
   `updated_at` to the current ISO8601 UTC timestamp, and saves to AsyncStorage.
3. The `useTodos` hook's state is updated via the `addTodo` callback so the list screen
   reflects the new item.
4. Dismiss the modal with `router.back()`.

No loading indicator is required for the create operation in v0.1.0; the AsyncStorage
write is fast.

### cancellation

A cancel button (or the system back gesture on Android) dismisses the modal without
creating a todo. The list is unchanged.

## edge cases summary

| scenario | expected behavior |
|---|---|
| empty title on submit | inline error; modal stays open; no todo created |
| whitespace-only title on submit | inline error; modal stays open; no todo created |
| title of exactly 200 chars | create succeeds; todo appears on list |
| title of 201 chars | inline error; modal stays open; no todo created |
| toggle on already-completed todo | flips back to incomplete; list updates |
| delete all todos | empty state is shown |
| AsyncStorage read fails on mount | error is caught and logged; list renders as empty |
| AsyncStorage write fails on mutation | error is caught and logged; in-memory state is still updated; no retry in v0.1.0 |
