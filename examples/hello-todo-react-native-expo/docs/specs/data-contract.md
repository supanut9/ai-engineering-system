# data contract — hello-todo-react-native-expo

This document describes the in-app data model and the AsyncStorage persistence schema for
`hello-todo-react-native-expo`. There is no HTTP API; the data contract is between the
in-memory repo (`lib/repo.ts`), the storage layer (`lib/storage.ts`), and the service
layer (`services/todos.ts`).

---

## entity: Todo

| field | type | required | default | notes |
|---|---|---|---|---|
| `id` | string | yes (service-assigned) | — | opaque string; generated as `Date.now().toString(36) + Math.random().toString(36).slice(2)`; unique within the session and across AsyncStorage |
| `title` | string | yes | — | non-empty after trim; max 200 characters; trimmed before storage |
| `completed` | boolean | yes | `false` | whether the todo is done |
| `created_at` | string | yes (service-managed) | current UTC time at creation as ISO8601 | immutable after creation |
| `updated_at` | string | yes (service-managed) | same as `created_at` at creation | updated on every successful toggle or title change |

The `Todo` TypeScript interface lives at `types/todo.ts`.

---

## validation rules

| field | rule |
|---|---|
| `title` | must be present; must be non-empty after trimming leading/trailing whitespace; trimmed value must be ≤ 200 characters |
| `id` | immutable after creation; generated by the service; never supplied by the caller |
| `created_at` | immutable; set by service at creation |
| `updated_at` | set by service on creation and on every successful mutation; not directly settable by the caller |
| `completed` | boolean only; no coercion from string or number |

## AsyncStorage schema

### key

```
@hello_todo:todos
```

A single key holds the entire todo list. This is appropriate for the scale of a reference
example with one entity and no pagination.

### value

A JSON-serialized array of `Todo` objects:

```json
[
  {
    "id": "lf3k2abc9xy",
    "title": "Buy milk",
    "completed": false,
    "created_at": "2026-05-16T10:00:00.000Z",
    "updated_at": "2026-05-16T10:00:00.000Z"
  },
  {
    "id": "lf3k2def7qr",
    "title": "Call dentist",
    "completed": true,
    "created_at": "2026-05-16T10:01:00.000Z",
    "updated_at": "2026-05-16T10:15:00.000Z"
  }
]
```

### read

`lib/storage.loadTodos()` calls `AsyncStorage.getItem('@hello_todo:todos')`. If the key
does not exist, returns `[]`. If the value fails `JSON.parse`, logs the error and returns
`[]` (graceful degradation — the user sees an empty list rather than a crash).

### write

`lib/storage.saveTodos(todos: Todo[])` calls
`AsyncStorage.setItem('@hello_todo:todos', JSON.stringify(todos))`. This is an
`async` function; callers `await` it before returning.

### clearing

`lib/storage.clearTodos()` calls `AsyncStorage.removeItem('@hello_todo:todos')`. Used in
test cleanup only; no UI exposes this operation in v0.1.0.

## in-memory repo (`lib/repo.ts`)

The module-level state is a `Todo[]` array. This is simpler than a `Map` because the
mobile example has no random-access-by-id in the storage layer — the full array is always
read and written atomically via AsyncStorage.

Exported functions:

| function | signature | notes |
|---|---|---|
| `setTodos` | `(todos: Todo[]) => void` | replaces the entire in-memory list; called on load |
| `getTodos` | `() => Todo[]` | returns a shallow copy of the list |
| `addTodo` | `(todo: Todo) => Todo[]` | appends and returns updated list |
| `updateTodo` | `(id, patch) => Todo[] \| undefined` | applies partial update; returns updated list or `undefined` if id not found |
| `removeTodo` | `(id) => Todo[] \| undefined` | removes by id; returns updated list or `undefined` if id not found |

All read operations return copies to prevent callers from mutating internal state.

---

## notes

- there is no separate "storage schema" and "API schema" — the same `Todo` interface is
  used in-memory, in AsyncStorage, and in the React component tree via `useTodos`
- all timestamps are ISO8601 strings (`new Date().toISOString()`) for JSON serialization
  consistency
- the `@hello_todo:todos` key prefix follows the AsyncStorage community convention of
  namespacing keys to avoid collisions with other libraries
